Test Plan for: Marvelious

Persona Type Tested: Primary

Context: Meeting Room
Duration: 5 minutes
User Consent: Victor's consent document
User Background Information: Victor's background information
Overall Objectives

We want to check the overall effectiveness of the Marvelious application by testing how fast different users can see the desired data: like characters, events, details. We want to find if the application is intuitive enough by identifing the obstacles of completing the below tasks. In this way we cand determinate how the user experience can be improved.

Tasks
  • Find "Hawkeye" comic book in the slide bar: 10 seconds;
  • Find "Incredible Hulk" comic book: 20 seconds;
  • Find related series to "IronMan" comic book: 35 seconds;
  • Filter character page by... : 15 seconds
  • Find the fifth incomming event: 15 seconds
Post-test Questions
  • Were the tasks hard?
  • Could you suggest other tasks?
  • What did you like?
  • What you didn't enjoy?
  • Will you take part in other case studies?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".
References
  • Fluid User Testing;
  • usability.gov Templates;
  • Handbook of Usability Testing by Jeffrey Rubin;
  • A Practical Guide to Usability Testing by Joseph S. Dumas and Janice C. Redish.
Other Notes

The user find all tasks quite simple, finishing most of them in half the time. However he specified that the carousel animation could be faster and that finding related comics of characters was not that intuitive.

His overall experience was good, but he would have appreciated much harders tasks.

He was more than happy to accept being called in further case studies.

Test Results for: Marvelious

Success Criteria

A successful design has been achieved when:

  • 100% of users should be able to find the comic book in the carousel
  • 100% of users should be able to find the comic book in the coresponding page
  • 50% of users should be able to find the related seris for a specified character
  • 100% of users should be able to use the filter function
  • 60% of users should be able to find the fift incomming event

Summary

List major findings in list or table format for quick and easy reading.

  • A number of 3 out of 3 users were able to find the comic book in the coresponding page;

  • A number of 2 out of 3 users were able to use the filter function;

Post-test Questions Answers

The tasks were pretty easy. We were suggested other tasks. Most of the userd enjoyed browsing the application and they will take part in next studies.

Test Observations

The tasks could have been harder. However, the good results satisfies success criterio on almost every task which means that the application is pretty intuitive. The opions gathered from the users were reflected and we come with some design improvements.

Potential Design Improvements

The carousel animation can be faster and the next button could not wait for the animation to finish in order to proceed to the next comic book

Related topics should be me more intuitive to find

Every event should have a date on it, to know the chronological order
